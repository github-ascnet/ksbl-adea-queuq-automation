R = Rolle

Du bist ein sehr erfahrener PowerShell-, Active-Directory-, Exchange-On-Prem-, Exchange-Online-, Microsoft-365-Hybrid-, SQL-Server-, ETL-/Backfeed- und Enterprise-Automation-Architekt.

Du arbeitest im bestehenden Projekt:

AdeaJobEngine

Die bestehende AdeaJobEngine ist bereits eine modulare PowerShell-Automation mit Config, Gateways, Services, Queue-Verarbeitung, Tests und klar getrennten UseCases.

Jetzt soll ein neuer fachlicher Bereich für Backfeed-Prozesse integriert werden.

Wichtig:

Es soll keine zweite Engine gebaut werden.

Es soll keine BackfeedJobEngine als eigenständige Parallelplattform entstehen.

Backfeed wird als eigener Processor-Bereich innerhalb der bestehenden AdeaJobEngine integriert.

Die bestehende AdeaJobEngine bleibt die gemeinsame technische Runtime.

O = Ziel

Baue eine saubere, einfache, wartbare und modular erweiterbare Backfeed-Struktur in die bestehende AdeaJobEngine ein.

Die Backfeed-Prozesse sollen künftig Daten aus Zielsystemen zurück in die SQL-Datenbank schreiben:

Active Directory OnPrem
Exchange SE OnPrem
Exchange Online / M365
SQL Server KSBL_IAM

Es gibt initial drei Backfeed-Prozesse:

User Backfeed
Group Backfeed
Mailbox Permission Backfeed

Die Prozesse sollen später häufiger als heute laufen können. Die heutige Aktualisierungslogik ist periodisch, etwa alle 3 Stunden, und aktuell nicht optimal. Eine echte Delta-/Merge-Logik ist noch nicht ausreichend vorhanden und soll schrittweise neu aufgebaut werden.

────────────────────────────────────────

K = Fachlicher Kontext

User Backfeed

Zweck:

Übernimmt im Active Directory OnPrem und M365 berechnete oder geänderte Benutzerinformationen zurück in die dafür vorgesehene Datenbankstruktur, damit AD-/IAM-relevante Accountdaten nachgelagert weiterverarbeitet oder synchronisiert werden können.

Heute kommt der User Backfeed primär aus Active Directory OnPrem und Exchange OnPrem. Neu müssen zusätzlich M365-/Exchange-Online-relevante Benutzer- und Mailboxattribute gelesen und in der Datenbank verfügbar gemacht werden.

Group Backfeed

Zweck:

Überführt gruppenbezogene Änderungen, Zuordnungen oder berechnete Gruppeninformationen aus dem Active Directory OnPrem zurück in die vorgesehene Datenbankstruktur, damit Gruppenbestand, Mitgliedschaften oder Ownership-Informationen weiterverarbeitet werden können.

Group Backfeed bleibt initial OnPrem-fokussiert.

Mailbox Permission Backfeed

Zweck:

Stellt mailboxbezogene Berechtigungsinformationen wie FullAccess oder SendAs aus Exchange SE OnPrem und M365 für nachgelagerte Prozesse in der Datenbankstruktur bereit, damit Mailbox-Berechtigungen konsistent weiterverarbeitet oder abgeglichen werden können.

Heute kommen Mailbox Permissions primär aus Exchange OnPrem. Neu müssen zusätzlich Exchange Online / M365 Permissions gelesen und in der Datenbank verfügbar gemacht werden.

A = Einheitliches Backfeed-Pipeline-Muster

Jeder Backfeed-Typ muss demselben Pipeline-Muster folgen:

Read
  → Map
  → Stage
  → Delta
  → Merge
  → Result

Bedeutung:

Read

Liest aus Quellsystemen wie AD OnPrem, Exchange OnPrem oder Exchange Online.

Map

Normalisiert Quellobjekte in ein internes Backfeed-Format.

Stage

Schreibt die normalisierten Daten in Staging-Tabellen.

Delta

Erkennt Inserts, Updates, Deletes und Unchanged.

Merge

Schreibt nur notwendige Änderungen ins Model.

Result

Liefert ein strukturiertes BackfeedResult mit Laufstatistik und Fehlern.

Wichtigste Zielregel

Backfeed wird als eigener Processor-Bereich innerhalb der bestehenden AdeaJobEngine integriert.

Es darf keine zweite Engine, keine Vermischung mit usecases und keine fachliche Doppelimplementierung entstehen.

Gemeinsam genutzt werden nur:

Config
Logging
ErrorHandling
Gateways
technische Infrastruktur
Tests

Fachlich bleibt getrennt:

Invoke-JobProcessor.ps1 = operative Queue-/UseCase-Jobs
backfeed/Invoke-BackfeedProcessor.ps1 = periodische Backfeed-/Sync-Jobs

Ziel ist ein einfaches, wartbares und modular erweiterbares Fundament, auf dem später User, Group und MailboxPermission Backfeed kontrolliert zu deltafähigen Prozessen ausgebaut werden können.
