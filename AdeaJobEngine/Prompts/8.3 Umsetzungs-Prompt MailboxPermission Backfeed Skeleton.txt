1. MailboxPermissionSourceReader.psm1 erstellen

Erstelle:

backfeed/MailboxPermission/MailboxPermissionSourceReader.psm1

Funktion:

Read-MailboxPermissionBackfeedSources

Parameter:

BackfeedContext

Aufgabe:

Orchestriert das Lesen aus den konfigurierten Quellen:

ExchangeOnPrem
ExchangeOnline

Für diese Phase:

Die Funktion darf Gateway-Funktionen aufrufen, falls sie bereits existieren.
Falls passende Gateway-Funktionen noch fehlen, keine produktiven Gateway-Implementierungen erfinden.
Stattdessen die SourceReader-Funktion so strukturieren, dass sie intern klar getrennte private/readbare Hilfsfunktionen für OnPrem und ExchangeOnline hat, die in Tests gemockt werden können.

Erwartetes Rückgabeformat:

Ein Array von rohen Permission-Objekten mit mindestens:

SourceSystem
PermissionType
MailboxIdentity
MailboxName
MailboxDistinguishedName
MailboxGuid
TrusteeIdentity
TrusteeName
TrusteeDomain
TrusteeDistinguishedName
TrusteeSid
AccessRights
IsInherited
Deny

Wichtig:

Keine echten Exchange-Cmdlets direkt im SourceReader aufrufen.
Keine Connect-ExchangeOnline-Aufrufe im SourceReader.
Keine New-PSSession-Aufrufe im SourceReader.
Keine direkten AD-Cmdlets im SourceReader.
SourceReader nutzt nur Gateways oder mockbare interne Funktionen.
Tests dürfen keine echten Zielsysteme erreichen.

2. MailboxPermissionMapper.psm1 erstellen

Erstelle:

backfeed/MailboxPermission/MailboxPermissionMapper.psm1

Funktion:

ConvertTo-MailboxPermissionBackfeedRows

Parameter:

RawPermissions

Aufgabe:

Normalisiert OnPrem- und EXO-Rohdaten in ein einheitliches DTO für Staging.

Ziel-DTO mit mindestens:

SourceSystem
PermissionType
MailboxKey
MailboxName
TrusteeKey
TrusteeName
TrusteeDomain
ObjectClass
AcePermissions
DistinguishedName
ExchHideFromAddressLists
AdReferenceObjectGuid
IsInherited
Deny
AccessRights
RowHash

Mapping-Regeln:

PermissionType muss FullAccess oder SendAs sein.
SourceSystem muss OnPrem oder ExchangeOnline sein.
MailboxKey bevorzugt MailboxGuid, sonst MailboxDistinguishedName, sonst MailboxIdentity.
TrusteeKey bevorzugt TrusteeSid, sonst TrusteeDistinguishedName, sonst TrusteeDomain + TrusteeName.
AcePermissions wird aus PermissionType oder AccessRights abgeleitet.
RowHash darf für diese Phase einfach über relevante Felder gebildet werden, sofern bereits BackfeedHash existiert. Wenn BackfeedHash noch nicht existiert, RowHash als TODO oder deterministischer einfacher String-Hash ohne neue Infrastruktur.
Keine unklaren Pflichtwerte erfinden.

Wichtig:

Mapper darf keine Zielsysteme aufrufen.
Mapper darf keine SQL-Verbindung verwenden.
Mapper ist reine Transformation.

backfeed/MailboxPermission/MailboxPermissionStagingWriter.psm1

Rückgabe mindestens:

Success
StagedCount
Message
ErrorCode

Wichtig:

Keine direkte Invoke-Sqlcmd-Nutzung im Writer.
Kein grosser INSERT-String pro Row in dieser Phase.
Keine TRUNCATE-/MERGE-Logik.
Keine History-Logik.
Keine produktiven SQL-Werte hardcoden.

4. MailboxPermissionBackfeedService.psm1 erweitern

Erweitere:

backfeed/MailboxPermission/MailboxPermissionBackfeedService.psm1

Funktion:

Invoke-MailboxPermissionBackfeed

Sie soll jetzt die Pipeline ausführen:

Read-MailboxPermissionBackfeedSources
  → ConvertTo-MailboxPermissionBackfeedRows
  → Write-MailboxPermissionBackfeedStaging
  → New-BackfeedResult

Verhalten:

Bei Erfolg:

Status = Succeeded
ReadCount = Anzahl RawPermissions
StagedCount = Anzahl staged Rows
FailedCount = 0
Errors = leer

Bei Fehler:

Status = Failed
FailedCount > 0
Errors enthält strukturierte Fehlermeldung
Bereits bekannte Zähler bleiben erhalten, soweit möglich

Wichtig:

Keine Delta-/Merge-Zähler setzen ausser 0.
Kein InsertedCount/UpdatedCount/DeletedCount berechnen.
Kein Merge ausführen.
Kein SQL-Refresh ausführen.
